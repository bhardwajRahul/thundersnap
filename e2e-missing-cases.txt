# E2E Test Coverage Gaps

## Rules for Adding Tests

* Use subagents and the task list whenever possible to avoid exhausting your context window.
* Tests should not themselves compile programs (let the makefile do it up front).
* Tests should not download from servers outside the local machine (construct test images locally instead).
* Tests should never have arbitrary timeouts; they cause flakes. Use channels or mutexes or
  some other sync primitive when possible, or in worst case, a polling loop
  with short timeouts and 'up to many' iterations that add up to a much longer one.
* **Timeout limits**: Each individual e2e test MUST complete in under 10 seconds. The entire
  e2e test suite MUST complete in under 30 seconds. Use `go test -timeout 10s` when running
  individual tests during development. Long timeouts (e.g., 10 minutes) hide hangs and bugs.
* Don't stop and ask questions along the way, I'm not watching you. Do your
  best and we'll fix it later. You can add a "notes for things to think
  about later" section at the end of this file if you want.
* Obviously you must run `make e2e` after each step. Yes, you have sudo access so this will work.

## Bug-Fixing Process

YOU WILL FIND BUGS while writing tests; that's the nature of e2e tests.
Fix them as you find them, but follow this process:

1. When an e2e test fails, add small unit tests (in pkg/*/foo_test.go) for each hypothesis
   until you find a failing one.
2. Don't delete "failed" hypotheses (tests that passed when you thought they'd fail) -
   keep them as additional coverage.
3. Fix the failing unit test, then re-run e2e to see if it fixes the whole bug. Repeat if not.
4. Commit the unit test(s) + bugfix together. If this also fixes the e2e test, include
   the e2e test in the same commit. If multiple mini-fixes are needed before the e2e passes,
   put the e2e test in a separate commit after the individual fixes.
5. As you complete new tests, check them off in this file in the same commit.

## Commit Workflow (jj)

* Each bug you find that needs a fix gets its own commit.
* If you add passing unit tests while investigating, batch them with the eventual fix.
* When building infrastructure (fake tsnet, OCI helpers), use `jj split` afterward to put
  infra in its own commit followed by the test that uses it.
* When done with a fix: `jj describe` it, then `jj new` so changes don't mix.

## Test Organization

* New e2e tests go in separate files by category: e2e/frame_test.go, e2e/snapshot_test.go, etc.
* Each section gets standalone TestXxx functions (1-3 per section), not subtests.
* Run the specific test while debugging, then `make test`, then e2e again before committing.

## Test Infrastructure Needed

* **Fake OCI tarballs**: Use a Go library if available, otherwise build helpers to construct
  valid OCI image tarballs locally for Docker import tests.
* **Fake mesh/tsnet**: Instantiate multiple thundersnapd instances in-process, listening on
  different local ports. Create a wrapper that supports both real tsnet and fake peer lists.
* **In-process thundersnapd**: For most tests, instantiate thundersnapd directly rather than
  managing subprocesses. Separate tests exist for packaging (container that runs thundersnapd).

## fidx-to-tsm Transition (DO THIS AFTER OTHER TESTS PASS)

See fidx-to-tsm-transition.md for full details. This transition replaces the legacy
fidx/mfidx format (SHA-1, 24-byte entries) with tsm/tsc (SHA-256, 40-byte entries).

**Prerequisites before starting transition:**
- [x] All Mesh/Content-Addressed tests pass (proves current download works)
- [x] All Snapshot Operations tests pass (proves snapshot creation works)
- [x] TSM/TSC Format tests pass (proves new format is correct)
- [x] Integration/Workflow tests pass (proves end-to-end works)

**Transition phases (do in order):**
- [x] Phase 1: Add TSC-based local deduplication (LoadLocalChunkMap in tsm package)
- [x] Phase 2: Implement TSM/TSC download path (tsm/download.go)
- [x] Phase 2b: HTTP serving of .tsm and .tsc files (already works - bupdateFileServer serves all files)
- [x] Phase 2c: Update download-snap to use TSM/TSC path (with fidx fallback)
- [x] Phase 2d: Test mesh download with TSM/TSC (run Mesh tests again)
- [x] Phase 3: Stop generating .fidx in thundersnapd (removed with bupdate deletion)
- [x] Phase 3b: Remove `ts fidx` command (removed with bupdate deletion)
- [x] Phase 4: Remove cmd/fidx and cmd/bupdate (done)
- [x] Phase 4b: Remove fidx generation code from bupdate/indexer.go (done - entire bupdate/ deleted)
- [x] Phase 4c: Update README.md to remove fidx references

**Legacy compatibility note:**
The fidx parsing code in bupdate/ has been removed. Old snapshots using fidx format
will need to be re-created with the new TSM/TSC format.
